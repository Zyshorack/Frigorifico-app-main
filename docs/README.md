# Documentacion

Esta carpeta guarda la documentacion viva de `Frigorifico API`.

Los documentos principales siguen un orden simple:

1. `01_objetivo.md`
2. `02_arquitectura.md`
3. `03_entidades_del_sistema.md`

Documento complementario:

- `endpoints_minimos_flujo_operativo.md`: flujo minimo para probar desde Swagger un circuito de frio, alerta y stock.
- `documentacion_tecnica.md`: explicacion extendida original para principiantes.

Seguimiento operativo:

- `issues/`: referencia local para ordenar el aprendizaje y relacionar la documentacion con endpoints concretos.

La documentacion debe estar en espanol, ser concreta y describir el estado objetivo del sistema desde ahora hacia adelante.
